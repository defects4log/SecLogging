Replication Package for FSE 2026 paper "Towards Secure Logging: Characterizing and Benchmarking Logging Code Security Issues with LLMs".

## Usage

Important files:

- The `data` folder contains our benchmark data: `SecLogging_Dataset.csv`: issue url for data extraction, `ground_Truth.csv` file for all ground truth values
- The `scripts` folder contains the all scripts in order to recreate the experiment
- The `results` folder contains all output files generated by scripts
- User run `code_change_extractor.py` to extract data from issue report url that is stated in `SecLogging_Dataset.csv` and it will generate `combined_result.csv` in result folder that contains all data extracted from the issue report. The extractor also writes intermediate JSON snapshots named *_before_edit.json and *_after_edit.json under `code_changes` folder



- Run `analyze_with_llm.py` that LLM will starts analyze code from `combined_result.csv` and generate `llm_analysis_results.csv` that has all the answer provided by LLM

```bash
python scripts/analyze_with_llm.py --combined-csv log_security/results/combined_result.csv --overview-csv "../data/ground_Truth.csv" --output log_security/results/llm_analysis_results.csv [--api-key "API KEY"] [--model xxx] [--batch-size xx] [--temperature x] [--use-description] [--use-explanation]
```

- --api-key: enter your api key in order to use the LLM
- --model: choose a model that you are willing to run
- --batch-size: reasonable batch size, for this experiment we recomment 5-10
- --temperature: set 1 as default
- --use-description: add this when you are willing to set task description as prompt hint
- --use-explanation: add this when you are willing to set category explanation as prompt hint

- --use-description and --use-explanation are optional command, you can use None/either/both

- Run `calculate_accuracy.py` to calculate categorize accuracy by LLM compare to ground truth

```
python scripts/calculate_accuracy.py --llm-results log_security/results/llm_analysis_results.csv --ground-truth "../data/ground_Truth.csv" --output log_security/results/accuracy_results.csv
```

- Run `merge_and_compare_similarity.py` to evaluate text and code similarity between ground truth fixed code and fixed code suggested by LLM

```
python scripts/merge_and_compare_similarity.py --llm-results results/llm_analysis_results.csv --combined-results results/combined_result.csv --output results/output_similarity_results.csv
```
